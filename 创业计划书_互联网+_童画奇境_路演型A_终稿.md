# 2. 项目背景与痛点（场景化）

## 2.1 儿童创意教育的“内容孤岛”与“成长不可视”

在当前的儿童美育与创意教育实践中，创作往往停留在“交作业”的终点：

- 作品停留在纸上/相册里，难以变成可持续互动的内容资产；
- 创意过程缺少结构化引导，孩子容易依赖模板照抄或直接放弃；
- 作品无法形成长期档案与能力画像，素质评价缺少证据链。

这会导致一个后果：家长与老师无法看到“孩子创作能力的变化曲线”，只能看到一张张孤立的作品。

## 2.2 家庭场景：亲子陪伴碎片化、共创缺失

家庭教育的真实矛盾是：家长愿意陪伴，但没有“可参与、可沉淀、可复盘”的载体。

- 亲子互动常被短视频与娱乐内容占据；
- 亲子创作缺少“共同完成一个作品”的结构化路径；
- 家长的参与很容易变成“指挥孩子画”，而不是共创。

童画·奇境把家长的参与设计成“配音共创”：家长以声音进入孩子作品，形成可回放、可纪念、可传播的家庭记忆资产。

## 2.3 学校场景：课堂创作难规模化、评价难量化

学校端（美术/语文/综合实践/STEAM）有天然需求：让孩子表达、叙事、协作。但现实中老师会被三件事卡死：

- 课堂时间有限，难以让每个孩子完成“画—讲—演”的完整表达链路；
- 作品难以统一展示与归档，班级管理成本高；
- 评价靠主观印象，缺少量化维度与可追溯材料。

童画·奇境提供教师端班级看板与能力分析，把“作品归档+成长评价”变成可规模化的学校服务。

## 2.4 行业根因：缺少“创作—引导—作品资产化—教学管理”的闭环平台

市面上常见方案要么是“单点生成工具”（生成图/生成文），要么是“内容平台”（看故事/看视频）。它们缺少四个关键要素的闭环：

1. 创作入口（画作/作品采集）
2. 引导过程（启发式而非替代式）
3. 作品资产化（可互动、可存证、可传播）
4. 教学管理（班级维度的归档与评价）

童画·奇境的价值就在于：把四件事打通，让创意教育从“活动”升级为“生态”。

# 3. 产品概述：童画·奇境如何闭环

## 3.1 产品定位（不是AI工具，是儿童创意教育生态）

童画·奇境不是“给孩子一键生成内容”的工具，而是把创作过程设计成“孩子主导、AI辅助、家庭共创、学校可管理”的闭环系统。

我们坚持三条产品原则：

- **保护原创力**：AI用启发式提问与创作引导，不替代孩子表达。
- **作品资产化**：每个作品都能变成可互动、可复盘、可纪念的内容资产。
- **成长可视化**：把作品与过程映射到能力维度与时间轴，形成可追溯档案。

## 3.2 用户与角色（孩子/家长/教师/学校）

- 孩子：创作者与导演（画→选分支→决定结局）
- 家长：共创者与见证者（配音、陪伴、分享）
- 教师：组织者与评价者（班级管理、能力分析、课堂落地）
- 学校/机构：采购方与推广方（素质教育数字化服务）

## 3.3 核心交互流程（与你现有系统一致）

核心流程与当前系统页面一致（可直接用于演示）：

1. 进入“奇境入口—3D漂浮展厅”（画廊）浏览作品
2. 点击“开始创作”进入“神笔工作台”
3. 上传画作
4. AI优化（增强色彩/细节）
5. AI动画生成（让画作动起来）
6. AI故事创作（并提供互动分支选择，多结局）
7. 进入作品详情页：原图/优化图/动画/互动故事
8. 家长进入“亲子配音室”录音或使用示例配音
9. 生成“区块链原创存证证书”（数字证书+版权教育）
10. 一键导出作品视频（用于分享/展示）
11. （学校端）教师在“班级管理”查看作品看板与能力分析

# 4. 核心创新与核心功能（6大功能全部实现）

> 本章为互联网+路演的“硬核部分”：每个功能都回答三个问题——它解决了什么痛点？如何形成闭环？为什么能规模化？

## 4.1 互动故事分支选择（3分支×2层深度=6种结局）

### 4.1.1 解决的痛点

传统故事阅读是单向输入，孩子的表达与决策能力训练不足；传统“写作文”难以让孩子在短时间内完成结构化叙事。

### 4.1.2 我们怎么做（产品机制）

- 作品生成“互动故事树”，每个节点提供多个选择；
- 孩子通过选择推动故事续写，形成不同结局；
- 系统记录“故事路径”（story_path），把孩子的决策过程沉淀为可复盘数据。

### 4.1.3 教育价值（评委加分点）

- 培养决策思维与因果推理
- 训练叙事结构与表达
- 在“选择—结果”的反馈中形成价值观引导

## 4.2 AI创作引导助手（启发式提问保护原创力）

### 4.2.1 解决的痛点

AI如果直接生成，会削弱孩子表达；但没有引导，孩子容易卡住。关键在于“启发式提问”。

### 4.2.2 我们怎么做

- 在创作流程关键步骤（优化/动画/故事）提供引导建议与鼓励；
- 引导内容以问题为主，帮助孩子补全世界观、角色动机、冲突与解决；
- 失败时自动降级为模板引导，保证稳定可用。

### 4.2.3 规模化逻辑

把老师/家长常见的启发式问题沉淀为模板库，形成可复用的“引导资产”。

## 4.3 成长时间轴档案（5维能力雷达图+里程碑）

### 4.3.1 解决的痛点

素质教育的评价缺证据，成长不可视；家校沟通缺共同语言。

### 4.3.2 我们怎么做

- 把作品与过程数据（创作次数、故事路径、配音参与、主题偏好等）映射到能力维度；
- 生成“成长时间轴”：每次创作成为一个里程碑；
- 以雷达图/趋势图呈现，形成可追溯档案。

> 注：能力维度与指标口径在本稿用`[需填入具体数据]`占位，后续可根据学校评价体系落地。

## 4.4 家长配音共创（亲子协同+录音/示例配音）

### 4.4.1 解决的痛点

家长想参与却缺少可沉淀的共创方式；亲子陪伴被碎片内容稀释。

### 4.4.2 我们怎么做

- 在作品详情页提供“亲子配音室”，支持录音与播放；
- 支持示例配音（不同角色语气），降低参与门槛；
- 配音与作品绑定，形成可回放的家庭记忆资产。

### 4.4.3 教育价值

- 情感纽带增强
- 口语表达与朗读训练
- 家庭共创形成积极反馈循环

## 4.5 区块链原创存证（数字证书+版权教育）

### 4.5.1 解决的痛点

儿童作品容易被忽视，缺少“原创意识”与“版权教育”；家长也缺少作品归属感。

### 4.5.2 我们怎么做

- 为作品生成“原创存证证书”（包含作品哈希、时间戳、作者信息等）；
- 通过区块链/可验证存证机制，让作品归属可追溯；
- 同时把“版权意识”融入儿童教育。

> 注：链上验证可做两种模式：
> - 演示/模拟模式（本地演示可用）
> - 真实链上模式（生成可查TxHash与浏览器链接）

## 4.6 班级教师管理（班级看板+能力分析，B端落地）

### 4.6.1 解决的痛点

学校端难以规模化组织创作与评价，作品归档与展示成本高。

### 4.6.2 我们怎么做

- 教师端统一管理班级作品
- 自动汇总班级创作数据与能力分布
- 支持教学活动的过程记录与成果展示

### 4.6.3 商业价值

教师端是学校采购的“抓手”，把项目从C端兴趣工具升级为B端可落地的教育服务。

- 完整功能模式：配置`DOUBAO_API_KEY`与`LLM_API_KEY`后启用真实模型能力。

## 5.3 数据与隐私保护（儿童数据最小化原则）

儿童数据合规是教育产品必须回答的问题。我们的基本原则：

- 最小化采集：只采集作品与必要的创作过程数据

- 可追溯审计：关键操作留痕（教师端、存证端）
